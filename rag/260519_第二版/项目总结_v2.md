# v2 景观搜图系统 — 项目总结

## 项目是什么

景观建筑设计师用的 AI 搜图系统。设计师用自然语言（"中式凉亭带水景"）搜索 NAS 上的 SU 模型预览图，系统返回相关图片和同名 ZIP 模型文件。

## 数据规模

- 1360 张 SU 模型预览图（知末来源，P01-P04 四个批次）
- 每张图有：同名 JPG + ZIP + 精标 JSON
- 存储在 NAS `\\192.168.1.203\知末备份`

## Pipeline（5 步）

| 步骤 | 脚本 | 输入 → 输出 |
|------|------|------------|
| Step01 | `step01_盲标_kimi_batch.py` | JPG → 盲标 JSON（tags + description） |
| Step02 | `step02_标签同义词归并.py` | 盲标标签 → Top500 规范词表 |
| Step03 | `step03_精标_kimi_batch.py` | JPG + 词表 → 精标 JSON（style/tags/plants/material/form/description） |
| Step04 | `step04_构建入库CSV.py` | 精标 JSON → RAGFlow 入库 CSV（path, filename, content） |
| Step05 | `step05_search.py` | 搜索入口：加载→翻译→打分→展示 |

## 搜索架构（当前版本）

### 三组件

```
Kimi k2.5  →  口语翻译 + 关键词提取（Top500 受控词表）
bge-m3    →  向量编码 + 全量打分（Xinference 本地部署）
RAGFlow   →  增量数据库 + 管理后台（不做检索）
```

### 启动：从 RAGFlow 全量加载

1. 调用 RAGFlow chunks 列表 API（不需要 question，不走检索过滤）
2. 解析每条 chunk：path, filename, tags
3. `doc_text = filename + " " + tags`
4. bge-m3 批量 embed → 1360×1024 矩阵存内存（5.5 MB）
5. 耗时 ~5 秒

### 查询：分层打分

```
用户说 "中式凉亭 水景"
  → Kimi: keywords=["中式","凉亭","水景"], vector_query="中式 凉亭 水景"
  → bge-m3 embed vector_query → 1×1024
  → 矩阵点积 → cosine × 100 → 1360 个基础分
  → 遍历：
      文件名命中 keyword → +300/词（按 _ 分词后逐段匹配）
      标签命中 keyword   → +100/词
  → 排序 → 全部命中 + 最多5条语义 → 分页30条展示
```

### 打分公式

```
最终分 = cosine(query_vec, doc_vec) × 100
       + Σ(filename命中keyword ? 300 : 0)
       + Σ(tags命中keyword ? 100 : 0)
```

### Kimi 翻译 Prompt 规则

- 风格未提则留空，不猜测
- 口语→规范词映射（"院子"→"庭院"）
- 复合词拆分去元标签（"奶油风格"→"奶油"，"风格"不进 keywords）
- vector_query = keywords 拼接，不加无关维度
- 用户原话实体词不能丢

### 结果截断

- 文件名或标签有 hit：全部显示
- 纯语义（无任何命中）：最多 5 条
- 每页 30 条，回车翻页，q 退出
- 2 字以下关键词不参与匹配

## 关键决策记录

1. **不用 RAGFlow 做检索** — 向量检索不可控，embedding 相似度不等于语义相关性
2. **不用 rerank（bge-reranker-v2-m3）** — 用户反馈精度差
3. **不用 LLM 裁判** — v1 验证后放弃，改用确定性规则
4. **文件名是最强信号** — 人工精选的关键词组合，+300 权重
5. **bge-m3 做基础分** — 语义兜底，精度够用
6. **启动预计算** — 查询只需一次 embed + 一次矩阵乘法（<1ms）
7. **chunks 列表 API 替代检索 API** — 不需要 question，不丢低相似度文档
8. **RAGFlow path 列加 P 前缀** — 防止 Table parser 把 `01` 解析成数字或日期

## 已知问题

- 2字短词可能误匹配（"客厅"∈"会客厅"），无词典无法根治
- Kimi 翻译不保证 100% 遵循 prompt 规则
- RAGFlow Docker 首次启动需 10-20 分钟（CUDA 依赖编译）

## 后续方向

- [ ] 精确匹配增强：2 字词做更严格的 `_`-segment 精确匹配
- [ ] Kimi 翻译质量：持续观察 bad case 调整 prompt
